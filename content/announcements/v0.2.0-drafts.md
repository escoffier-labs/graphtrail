# GraphTrail v0.2.0 announcement drafts (NOT posted)

## X / Mastodon

GraphTrail v0.2.0 is out, and it's the first version I'd actually call a code graph.

Call resolution is now qualifier and import aware: `a.save()` resolves through the module you imported, not to every same-named function in the repo. On a 4.6k-symbol codebase that deleted two thirds of the edges. All of them were false positives.

Also new: `impact --depth` for real blast radius (transitive callers, cycle-safe), `neighbors` for a file's structural neighborhood, path-scoped search, and a `repos` tool so an MCP agent can discover what it can query.

First crates.io release: cargo install graphtrail

https://github.com/escoffier-labs/graphtrail

## LinkedIn (longer)

Shipped GraphTrail v0.2.0 tonight.

GraphTrail is a local code-graph sidecar for AI coding agents: it parses a repo with tree-sitter into a SQLite graph of symbols, imports, and call edges, then answers "who calls this", "what breaks if I change it", and "what should I read before editing" over a CLI and a read-only MCP server. No daemon, no network, one file per repo.

The headline in 0.2.0 is precision. Version 0.1.0 resolved calls by bare name, so `handoff.lint()` and `skills.lint()` pointed at each other's functions. Resolution is now qualifier and import aware, including relative Python modules and Rust crate paths. On a 4.6k-symbol repo the edge count dropped from 29k to under 10k with identical symbol coverage. Every removed edge was noise an agent would have chased.

Also new: transitive impact with hop counts (`impact <symbol> --depth 3`), file neighborhoods, path-scoped symbol search, richer freshness stats, and a first crates.io release, so it's now just `cargo install graphtrail`.

The part I'm most excited about is the loop it closes: my agent orchestration now attaches a GraphTrail context pack to every dispatched worker's brief. In testing, a planner named the exact file and private functions a task involved before any worker opened a single file.

https://graphtrail.escoffierlabs.dev
